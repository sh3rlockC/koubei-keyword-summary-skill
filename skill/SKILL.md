# 口碑关键词与摘要提炼

目标：读取已有口碑 Excel，分别对 `最满意` 与 `最不满意` 提炼高频关键词、方向摘要、代表性原句，并导出 Excel。

## 1. 输入要求

优先接受：

- 已导出的口碑 Excel 文件
- 明确车型名（用于输出命名）
- 若用户指定分析方向，也可限定在空间、续航、智能化、底盘、配置、价格等方向

默认兼容两类表结构：

- 单 sheet：同时包含 `最满意`、`最不满意`
- 多 sheet：如 `购车口碑`、`试驾口碑`

## 2. 提炼目标

至少输出四层结果：

1. **关键词频次**
   - 分别统计 `最满意` / `最不满意`
   - 以“方向词”为主，不追求分词学术精度
   - 应合并同义表达，例如：
     - `空间大` / `空间宽敞` → `空间`
     - `车机卡` / `智能化一般` / `软件不好用` → `车机智能化`
   - 默认输出 **三口径统计**：
     - `主方向口径`：每条口碑只归到 1~3 个主方向，最接近人工阅读习惯
     - `条目口径`：一条口碑命中某方向记 1 次，适合稳定汇报
     - `语句归并口径`：拆句后对相似表达归并计数，适合深入分析
   - 不采用手工固定口径覆盖统计结果；若做人工修正，必须单独标注
   - 必须清理口头词、套话、空泛词，如 `哈哈`、`总的来说`、`感觉`、`整体` 等

2. **方向级摘要**
   - 例如：空间、配置性价比、续航能耗、座椅舒适性、底盘悬架、外观设计、车机智能化、辅助驾驶、充电补能、隔音静谧性、储物/实用性
   - 每个方向给出：
     - 提及次数
     - 简短摘要
     - 代表性原句 3-5 条
   - 代表性原句尽量截到 1-2 句，避免整段长评原样灌入摘要表

3. **总览摘要**
   - 输出适合直接发给业务/产品的简版结论
   - 分为：
     - 最满意 Top N 方向
     - 最不满意 Top N 方向
     - 一句话车型印象

4. **业务摘要 / 汇报版摘要**
   - 输出更像分析结论而不是统计表
   - 至少包含：
     - 核心卖点
     - 核心槽点
     - 产品改进建议
     - 适合人群 / 使用场景（如能从口碑中稳定看出）

## 3. 推荐方向词典

默认优先聚类到以下方向：

- 空间
- 座椅舒适性
- 底盘悬架
- 动力响应
- 续航能耗
- 充电补能
- 外观设计
- 内饰质感
- 车机智能化
- 辅助驾驶
- 配置性价比
- 隔音静谧性
- 储物/实用性
- 后排头部空间
- 品控/做工

如果遇到明显高频新方向，可以追加，但不要把标签打得过碎。

## 4. 输出文件规则

默认输出一个 Excel，建议包含以下 sheet（按交付顺序排列）：

- `总览摘要`
- `业务摘要`
- `产品机会点`
- `最满意方向统计`
- `最不满意方向统计`
- `最满意方向摘要`
- `最不满意方向摘要`
- `最满意主题`
- `最不满意主题`
- `最满意高频词`
- `最不满意高频词`

如用户需要，也可附带 JSON 侧产物。

默认命名示例：

- `车型名_口碑关键词摘要.xlsx`
- 例如：`启源A06_口碑关键词摘要.xlsx`

## 5. 抽取原则

- 不改写用户原句的事实含义
- 关键词提炼优先服务业务阅读，不追求 NLP 花活
- 同义归并要稳定、一致
- `代表性原句` 尽量保留原文口语感
- 如果某方向只出现 1-2 次，也可以保留，但不要夸大为主流结论

## 6. 推荐执行方式

优先使用附带脚本：

- `scripts/summarize_koubei_excel.py`

示例：

```bash
python3 skills/koubei-keyword-summary/scripts/summarize_koubei_excel.py \
  --input /path/to/koubei.xlsx \
  --output /path/to/启源A06_口碑关键词摘要.xlsx \
  --model-name 启源A06
```

## 7. 校验要求

导出前至少检查：

- 是否识别到 `最满意` 和 `最不满意` 两列
- 是否成功读取全部 sheet
- 关键词统计非空
- 总览摘要非空

如果原始 Excel 缺列或为空，直接报错，不输出伪结果。
